计算机教育中缺失的一课

课程概览与shell

Q：什么是shell？

A：计算机有多种交互接口供用户输入指令，但图像用户界面(Graphical User Interface, GUI)的存在限制了指令的输入，因为不可能点击不存在的按钮或语音输入不存在的指令，因此使用最原始或者说最根本的方式，文字接口：Shell

Bourne Again SHell, 简称 “bash” ，是使用最广泛的一种shell

输入命令，之后被shell解析，shell基于空格分割命令并进行解析，与Python等一样，shell是一个编程环境，在shell中执行命令，实际上是在执行一段shell可以解释执行的代码，如果指令不是shell的编程关键字，会在环境变量$PATH中找

Shell工具和脚本

Q：shell脚本是什么？

A：shell就类似或者说就是一门编程语言，和C++、Python一样，有自己的语法，关键字等，使用shell编写程序实现的一些简单的功能，就是shell脚本
shell脚本使用领域与C++等不同，shell用于Linux上一些小功能的实现，我个人理解是为了省去逐条命令会产生的中间文件，直接将多个步骤产生的输出作为下一个输入，实现一个完成的功能，不必一条命令一条命令进行操作

shell工具

一些方便日常使用的工具，例如find工具用来查找，与其它命令融合完成工作

编辑器(Vim)

写作和写代码之间的区别是很大的，就像word 和 VSCode的区别一样，写代码经常在多个文件之间切换，修改，而不会连续编写一大段文字

Vim避免使用鼠标，方向键等，熟练以后能大幅提升效率
Vim的界面本身就是一个程序语言，键入操作就是命令
Vim可通过修改配置文件自定义和扩展Vim

数据整理

将某种格式存储的数据转换成另一种格式

通常借助正则表达式和其它工具配合，以达到想要的效果，例如 sed、awk 等工具

实际就是通过各种工具进行配合，实现想要对数据进行的操作，例如
从日志中找到连接服务的用户，同一个用户多条记录合并，结果按次数排序

命令行环境

任务控制

通过信号机制执行进程间的通信，一个进程收到信号后会有以下行为，停止执行、处理信号、基于信号传递的信息改变执行过程

ctrl+c 可结束进程，是通过发送信号到进程实现的，进程接收到后终止
ctrl+z 可暂停进程，fg 和 bg 可分别在前台和后台恢复暂停的工作
jobs 可列出当前终端中未完成的全部任务，可使用 pid 引用这些任务

终端多路复用

不打开新的终端窗口，而是分割出多个终端窗口，同时执行多个任务

tmux 是目前最流行的终端多路复用器

难度不高，只需多使用，熟练即可

别名

频繁输入一长串命令过于麻烦，因此可将长命令起别名，直接使用别名即可

常使用 alias 起别名，格式为 alias 别名="命令内容" ，禁用使用unalias
注意，= 两边没空格，且 alias 只接受一个参数

默认情况下，shell不保存别名，若要持续生效，需要将配置放进shell的启动文件

配置文件

很多程序的配置文件都是纯文本格式，因以.开头，成为点文件，默认是隐藏文件

对于bash来说，可通过编辑.bashrc文件实现，可根据自身需求修改配置文件，启动时shell程序会加载配置项

配置文件有个问题，可能不能在不同系统或shell上生效，所以可根据需求，针对不同设备编写不同配置，通过if实现

远端设备

SSH，Secure Shell，安全shell，在不安全的网络上用于安全的远程登录

连接其它服务器格式为：ssh user_name@server
指以用户名 user_name 登录服务器 server，服务器可通过URL或IP指定，例如：
ssh foo@bar.mit.edu (URL指定)、ssh foobar@192.168.1.42 (IP指定)

为避免每次登录都输入密码的麻烦，SSH密钥使用了公钥，即只需要向服务器证明客户端持有对应私钥即可，而不必公开私钥
私钥等同于密码,一般为~/.ssh/id_rsa 或 ~/.ssh/id_ed25519
ssh 会查询 .ssh/authorized_keys 来确认那些用户可以被允许登录

版本控制

版本控制系统(VCS)是一类用于追踪源码，文件等改动的工具

Git的数据模型只包括两部分：tree(树，指目录) blob(数据，指文件)
数据对象有三种：blob，tree，commit

Git仓库：对象 和 引用

强烈推荐 Pro Git 作为学习Git的教材

调试及性能分析

调试代码

可使用多种方式调试程序，输出语句、日志、调试器
一些情况下需要查看网络数据包才能定位问题，可使用 Wireshark 工具
静态分析工具可在不运行程序情况下分析出一些简单的问题

性能分析

通过计算程序前后花费的时间进行分析

使用一些性能分析工具进行性能分析
例如cpu和内存

其他的完全看不懂，基础再学点回头再看…

元编程

项目有完整的流程，而非写完代码即可

构建系统

大多数系统，都有一个 构建过程 ，要执行一系列操作，可通过工具实现

这些工具本质上很相似，只需要定义 依赖、目标、规则，即：

您必须告诉构建系统您具体的构建目标
系统的任务则是找到构建这些目标所需要的依赖
并根据规则构建所需的中间产物，直到最终目标被构建出来

make 是最常用的系统之一，对于中小型项目来说足够
当执行 make 时，会参考当前目录下名为 Makefile 的文件，所有构建目标、相关依赖和规则都需要在该文件中定义

依赖管理

就构建的项目而言，依赖可能是某些程序(例如 python)、系统包 (例如 openssl)或相关编程语言的库(例如 matplotlib)

对于依赖，需要关注依赖的版本号，为防止依赖不可用，要注意版本控制

版本号一般格式为：主版本号.次版本号.修订号，版本号递增规则如下：

  1. 主版本号：当你做了不兼容的 API 修改
  2. 次版本号：当你做了向下兼容的功能性新增
  3. 修订号：当你做了向下兼容的问题修正

先行版本号及版本编译信息可以加到“主版本号.次版本号.修订号”的后面，作为延伸

可以理解为：

  • 如果新的版本没有改变 API，请将补丁号递增；
  • 如果您添加了 API 并且该改动是向后兼容的，请将次版本号递增；
  • 如果您修改了 API 但是它并不向后兼容，请将主版本号递增。

持续集成系统

随着项目规模增大，修改代码之后还有很多额外工作，例如上传一份新版本的文档、上传编译后的文件到某处、发布代码到 pypi，执行测试套件等等

持续集成(CI)指的就是 代码变动时，自动运行的东西，市场上有很多CI工具存在

CI的工作原理基本一致，即：
用户在代码仓库中添加一个文件，描述当前仓库发生任何修改时，应该如何应对

大杂烩

修改键位映射

可对键盘上的键触发的事件进行修改，改变键盘按键的键位映射

可使用 AutoHotkey 等工具实现

守护进程

大部分计算机都有一系列在后台保持运行，不需要用户手动运行或者交互的进程。这些进程就是守护进程

API(应用程序接口)

可通过编程方式来访问提供API的服务的数据

VPN

使用VPN后，发出的流量看上去来源于 VPN 供应商的网络而不是“真实”地址，同时你实际接入的网络只能看到加密的流量

实际上使用 VPN 只是把原本对网络供应商的信任放在了 VPN 供应商那里——网络供应商能看到的，VPN 供应商也都能看到

开机引导及Live USB

计算机启动时，BIOS 或 UEFI 会在加载操作系统前对硬件系统初始化，称为引导

可在引导菜单中选择硬盘以外的设备加载操作系统，例如 Live USB

Live USB 是包含完整操作系统的闪存盘，用于安装系统，修复数据等多种用途

Docker, Vagrant, VMs, Cloud, OpenStack

虚拟机 以及 容器化 等工具可以帮助，模拟一个包括操作系统的完整计算机系统
虚拟机 可用于创建独立的测试或开发环境，以及安全测试的沙盒

虚拟机 和 容器 的区别：
虚拟机会执行整个OS栈，包括内核(即使和主机内核一致)；
容器避免运行其它内核实例，与主机分享内核；隔离性较弱，只有在与主机运行相同内核时才能正常工作；
因此，容器的开销小于完整的虚拟机

交互式笔记本编程

交互式记事本 可以帮助开发者进行与运行结果交互等探索性的编程。现在最受欢迎的交互式记事本环境大概是 Jupyter
